CADAgent vs ChatGPT for CAD

Many engineers have tried asking ChatGPT to make CAD models. You type something like "create a flanged bracket with four bolt holes," and you get back... a paragraph of text. Maybe some OpenSCAD code. Maybe an STL file if you push hard enough. The results are interesting, but they're not what you'd actually use in a production workflow. Here's why that gap exists, and what a purpose-built tool does differently.

What happens when you ask ChatGPT to make a CAD model

ChatGPT is a general-purpose language model. It knows a lot about CAD concepts and can discuss geometry intelligently, but it doesn't run inside any CAD environment. When you ask it to "make" a model, what you actually get falls into a few categories:

  • OpenSCAD or CadQuery scripts that you then have to copy, paste into another application, debug, and render yourself.
  • Descriptions of geometry that read well but don't translate directly into features you can import.
  • STL-generating code that produces mesh output — triangulated surfaces with no underlying intelligence.

None of these give you a native parametric model. There's no timeline, no feature tree, no sketch constraints, and no way to go back and change a dimension without regenerating everything from scratch. You're essentially getting a recipe and being told to cook it yourself.

The parametric difference

This matters more than it might seem at first. In real engineering workflows, the initial shape is only the beginning. You need to modify fillet radii after running stress analysis. You need to adjust wall thickness when the material spec changes. You need to suppress a feature to create a simplified version for a different manufacturing process. Parametric models handle all of this because every feature knows what it depends on and how it was created.

Mesh files — STLs, OBJs, 3MFs — are what the industry calls dead geometry. They represent a shape as a cloud of triangles with no knowledge of how that shape was made. You can't edit a fillet radius on an STL because the fillet doesn't exist as a concept; it's just triangles that happen to form a curved surface. Try changing a bolt hole from M6 to M8 on a mesh file and you'll understand the problem immediately.

For prototyping and 3D printing, mesh files can be fine. But for anything headed toward CNC machining, injection molding, or sheet metal fabrication, your manufacturer needs parametric or at minimum STEP/BREP geometry. The model has to be more than a shape — it has to encode design intent.

How CADAgent solves this

CADAgent takes a fundamentally different approach. Instead of generating code that you run elsewhere, it operates directly inside Autodesk Fusion 360 as a native add-in. When you describe geometry in natural language, CADAgent translates your description into actual Fusion 360 API calls that create real parametric features.

The result is a proper feature chain: a sketch gets created on a plane, an extrusion references that sketch, a fillet references edges on that extrusion. Every feature appears in your Fusion 360 timeline and is fully editable. You can double-click any feature to change its dimensions, reorder operations, or suppress features — exactly as if you'd created them manually.

Under the hood, CADAgent uses Anthropic's Claude (not ChatGPT) as its language model, specifically tuned for Fusion 360 API generation. The distinction matters: rather than producing generic geometry descriptions, the model outputs precise API calls that Fusion 360 understands natively. The result is real sketches with constraints, extrusions with properly defined extents, fillets with editable radii, and patterns with adjustable counts.

When to use which

ChatGPT and CADAgent aren't really competitors — they solve different problems. Here's a practical breakdown:

  • Use ChatGPT when you want to learn CAD concepts, brainstorm design approaches, generate OpenSCAD or CadQuery scripts for quick visualizations, or get help understanding manufacturing processes. It's an excellent engineering assistant for knowledge work.
  • Use CADAgent when you need to create actual parametric geometry inside Fusion 360 that you'll refine, document, and manufacture. When the output needs to be a real, editable CAD model — not a script or a mesh — that's what CADAgent is built for.

The best workflow for many engineers is using both: ChatGPT to think through a design problem and explore options, then CADAgent to build the actual geometry once you know what you want. Different stages of the design process call for different tools.
